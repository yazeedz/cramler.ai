# Navigation & Sidebar

## User Context Model

Each user is attached to:
- **One organization** at a time
- **One brand** within that organization

The app view is always scoped to: `Organization / Brand`

Users can switch between organizations and brands via the header.

## Context Persistence

When user switches organization or brand:
1. Save selection to `user_preferences` or local storage
2. Reload page data for new context
3. Update URL if needed (e.g., `/products` stays same, data changes)

```typescript
// Store current context
interface UserContext {
  organizationId: string;
  brandId: string;
}

// In layout or provider
const { organizationId, brandId } = useUserContext();

// All data fetches use this context
const products = await getProducts(brandId);
```
